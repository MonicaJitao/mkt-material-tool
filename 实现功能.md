## 一、项目概述

**Capybara** 是一个基于 Electron + React + TypeScript 的桌面应用，定位为 **AI 协作文档预览与终端浏览器**。核心理念是"零配置"——AI 助手创建或修改文件后，Capybara 自动检测并预览，无需任何额外命令。

### 技术栈


| 层级          | 技术                                     |
| ----------- | -------------------------------------- |
| 桌面框架        | Electron 39.2.7                        |
| 前端框架        | React 18.3.1 + TypeScript 5.7.2        |
| 构建工具        | Vite 6.0.7                             |
| 状态管理        | Zustand 5.0.3                          |
| UI 组件库      | Ant Design 6.2.1                       |
| 终端渲染        | xterm.js 5.3.0                         |
| 终端进程        | node-pty 1.1.0                         |
| Markdown 解析 | marked 17.0.1                          |
| 文件监听        | chokidar 5.0.0                         |
| OCR 识别      | @gutenye/ocr-node 1.4.8                |
| 背景移除        | @imgly/background-removal-node 1.4.5   |
| 国际化         | i18next 23.16.0 + react-i18next 15.2.3 |
| PNG 导出      | html-to-image 1.11.13                  |
| 测试          | Vitest 3.0.6                           |

## 二、已实现功能点详解

### 1. 多标签页文件管理

**实现位置：** `src/renderer/stores/tabStore.ts` + `src/renderer/components/TabBar/`

- Chrome 风格标签页，支持拖拽排序
- 支持文件类型：`.md`、`.html/.htm`、`.png/.jpg/.jpeg/.gif/.bmp/.webp/.svg`、`.pdf`
- 智能标签复用：新文件会复用欢迎页标签，而不是无限新开
- 画廊模式（Gallery Mode）：缩略图网格视图浏览所有打开的文件
- 标签历史与最近文件记录

---

### 2. 集成终端

**实现位置：** `src/main/terminalManager.ts` + `src/renderer/components/Terminal/`

- 基于 **xterm.js** 渲染终端 UI，**node-pty** 管理真实 shell 进程
- 可停靠位置：左侧、右侧、浮动窗口
- 目录跟踪：监听 `cd` 命令，自动更新文件监听目录
- 键盘快捷键：`Ctrl+T` 新建标签、`Ctrl+W` 关闭、`Ctrl+Tab` 切换
- 滚动缓冲区限制 1000 行

---

### 3. 文件自动监听与预览

**实现位置：** `src/main/fileWatcher.ts`

- 基于 **chokidar** 监听当前目录
- 新建 `.md`/`.html` 文件 → 自动在新标签页打开
- 已打开的文件被修改 → 自动刷新预览
- 防重复事件去抖动处理
- 安全限制：拒绝监听系统目录、超过 500 个文件或 200 个子目录的大目录
- 可通过终端头部开关切换

---

### 4. Markdown 查看器

**实现位置：** `src/renderer/components/Viewers/MarkdownViewer/`

- 基于 **marked.js** 实时渲染 Markdown
- 支持 GFM（GitHub Flavored Markdown）
- 代码块语法高亮（支持 180+ 语言）
- 编辑模式：直接修改源码并保存

---

### 5. HTML 可视化编辑器（核心功能）

**实现位置：** `src/renderer/components/Viewers/HtmlViewer/`

这是项目最复杂的模块，采用 **iframe 双向消息通信架构**：

```
React (HtmlViewer)
  ↕ postMessage
iframe（注入了编辑器 JS）
```

#### 5.1 元素选择与高亮

- 点击预览区域中的任意元素即可选中
- 选中元素高亮显示

#### 5.2 样式面板（右侧边栏）— `StylePanel/`


| 分区             | 控制项                                                      |
| -------------- | -------------------------------------------------------- |
| 布局（Layout）     | display、flexDirection、justifyContent、alignItems、gap、flex |
| 间距（Spacing）    | 可视化盒模型，独立控制 margin/padding 四边                            |
| 排版（Typography） | 字体大小、字重、颜色、对齐方式、行高                                       |
| 边框（Border）     | 宽度、样式、颜色、圆角                                              |
| 背景（Background） | 背景色、背景图                                                  |
| 导出（Export）     | 将选中元素导出为 PNG 图片                                          |


#### 5.3 DOM 树导航（左侧边栏）— `NodeTree/`

- 可视化展示 DOM 层级结构
- 点击节点选中对应元素

#### 5.4 拖拽移动

- 在预览中拖拽元素改变位置

#### 5.5 撤销/重做 — `hooks/useEditorHistory.ts`

- 完整的编辑历史，500ms 防抖记录

#### 5.6 图片路径处理 — `utils/injector.ts`

- 本地图片路径 → base64 data URL（用于 iframe 显示）
- 保存时还原为原始路径

---

### 6. 图片查看器

**实现位置：** `src/renderer/components/Viewers/ImageViewer/`

- 图片展示（base64 data URL）
- **OCR 文字识别**：基于 `@gutenye/ocr-node`，提取图片中的文字
- 提取结果一键复制到剪贴板
- **背景移除**：基于 `@imgly/background-removal-node`（AI 模型）

---

### 7. PDF 查看器

**实现位置：** `src/renderer/components/Viewers/PdfViewer/`

- 嵌入浏览器原生 PDF 渲染器

---

### 8. 国际化（i18n）

**实现位置：** `src/renderer/i18n/`

- 基于 **i18next + react-i18next**
- 支持中文/英文切换
- UI 内语言切换器组件

---

### 9. 主题支持

- 亮色/暗色主题切换
- 持久化主题偏好

---

### 10. 单实例控制

**实现位置：** `src/main/index.ts`

- 防止多个应用实例同时运行
- 通过 socket 通信将 URL 路由到已有实例

---

### 11. MCP Server 集成

**实现位置：** `src/mcp-server/` + `src/main/htmlExportServer.ts`

- 实现 **Model Context Protocol** 服务端
- 允许 AI 助手通过 MCP 协议导出 HTML 文件
- Capybara 与 AI 工具链深度集成的关键

---

### 12. 状态管理（Zustand）

**实现位置：** `src/renderer/stores/`


| Store              | 管理内容              |
| ------------------ | ----------------- |
| `tabStore.ts`      | 标签页列表、活跃标签、画廊模式   |
| `terminalStore.ts` | 终端可见性、文件监听状态、停靠位置 |
| `toastStore.ts`    | 全局通知消息            |


---

### 13. IPC 通信通道


| 通道                  | 方向              | 用途       |
| ------------------- | --------------- | -------- |
| `terminal-start`    | Main → Renderer | 启动终端     |
| `terminal-input`    | Renderer → Main | 发送终端输入   |
| `terminal-output`   | Main → Renderer | 终端输出数据   |
| `terminal-resize`   | Renderer → Main | 调整终端大小   |
| `load-file`         | Main → Renderer | 在标签页加载文件 |
| `read-file`         | Renderer → Main | 读取文件内容   |
| `write-file`        | Renderer → Main | 写入文件内容   |
| `file-watch-toggle` | Renderer → Main | 切换文件监听   |
| `show-toast`        | Main → Renderer | 显示通知     |

## 三、HTML 编辑开源方案调研

### 方案对比


| 方案            | Stars | License | 定位               | React 支持          | 适合 Capybara |
| ------------- | ----- | ------- | ---------------- | ----------------- | ----------- |
| **GrapesJS**  | 25.7k | BSD-3   | 完整 HTML 页面可视化构建器 | `@grapesjs/react` | ⭐⭐⭐⭐⭐       |
| **Tiptap**    | 36.5k | MIT     | 无头富文本编辑器         | 原生支持              | ⭐⭐⭐         |
| **Craft.js**  | ~12k  | MIT     | React 拖拽页面编辑框架   | React 原生          | ⭐⭐⭐⭐        |
| **Puck**      | -     | MIT     | React 可视化编辑器     | React 原生          | ⭐⭐⭐⭐        |
| **Quill**     | -     | BSD     | 富文本内容编辑          | 社区支持              | ⭐⭐          |
| **Lexical**   | -     | MIT     | 无头文本编辑框架         | React 原生          | ⭐⭐          |
| **BlockNote** | -     | MIT     | 块编辑器（基于 Tiptap）  | React 原生          | ⭐⭐          |


> **注意：** 富文本编辑器（Tiptap/Quill/Lexical）定位是编辑内容片段，输出 HTML 片段；可视化页面构建器（GrapesJS/Craft.js/Puck）定位是编辑完整 HTML 页面结构。Capybara 的场景是后者。

---

### 方案 A：GrapesJS（最推荐）

**GitHub：** [https://github.com/GrapesJS/grapesjs](https://github.com/GrapesJS/grapesjs)

- **License：** BSD 3-Clause（完全开源免费）
- **React 集成：** 官方提供 `@grapesjs/react` 包
- **核心能力：**
  - 拖拽式块管理（Block Manager）
  - CSS 样式管理器（Style Manager）—— 与现有 StylePanel 功能高度重叠
  - 图层管理器（Layer Manager）—— 与现有 NodeTree 功能高度重叠
  - 代码查看器（HTML/CSS 双栏）
  - 资源管理器（Asset Manager）
  - 40+ 官方/社区插件
- **优点：** 功能最完整，与 Capybara 现有 HTML 编辑需求高度匹配，可直接替换自定义 iframe 方案，大幅减少自定义代码量
- **缺点：** 体积较大，学习曲线较陡，需要较多集成工作

---

### 方案 B：Tiptap（适合升级 Markdown 编辑器）

**GitHub：** [https://github.com/ueberdosis/tiptap](https://github.com/ueberdosis/tiptap)

- **License：** MIT（核心免费，Pro 扩展收费）
- **核心能力：** 无头架构，100+ 扩展，基于 ProseMirror，支持协作编辑
- **适用场景：** 更适合将 Markdown 编辑器升级为 Notion 风格块编辑器，而非 HTML 页面可视化编辑

---

### 方案 C：Craft.js（React 原生页面构建框架）

**GitHub：** [https://github.com/prevwong/craft.js](https://github.com/prevwong/craft.js)

- **License：** MIT
- **特点：** 专为 React 设计，用自己的 React 组件构建拖拽编辑器
- **适用场景：** 如果重构后想用 React 组件作为 HTML 模板单元

---

### 方案 D：Puck（React 可视化编辑器）

**GitHub：** [https://github.com/measuredco/puck](https://github.com/measuredco/puck)

- **License：** MIT
- **特点：** 开源 React 可视化编辑器，可嵌入任何应用，与现有 React 组件无缝集成
