## 人工验收案例

使用以下 Brief：

- 活动主题：造城者
- 风格：明亮摄影融合
- 城市：深圳或北京
- 底图要求：城市地标 + 产业氛围，无文字
- HTML 要求：文字可读，底图正确，保存版本可回看

验收步骤：

- [ ] 选择“城市产业海报”模板
- [ ] 创建活动，项目名可填“微众银行营销素材”，活动名可填“五一劳动节造城者”
- [ ] 填写城市、行业、目标客群、主题、客户经理、公司、视觉风格、尺寸
- [ ] 生成视觉方案，确认主题为“造城者”，风格为“明亮摄影融合”
- [ ] 确认方案后创建至少 4 张底图候选
- [ ] 确认每个底图槽位都有独立进度、完成或失败状态
- [ ] 选择一张无文字、包含城市地标和产业氛围的底图
- [ ] 生成 HTML 海报
- [ ] 检查 HTML 预览中文字可读，底图为刚选择的图片
- [ ] 编辑 HTML 后保存为新版本
- [ ] 在版本历史中切回旧版本，确认旧版本仍可回看且未被覆盖

## 当前依赖其它 Agent 的部分

- Claude 与 Tuzi 的真实网络调用质量仍依赖 Agent B / Agent C 的 Provider 配置、API Key 和中转站稳定性。
- HTML 视觉质量、文字可读性和地标准确性仍依赖 PromptAgent / Claude 输出质量，自动化测试只覆盖契约和安全底线。
- 前端完整 Playwright 交互测试尚未接入，目前交付的是无新增依赖的基础冒烟测试。

## 建议修复顺序

1. 先运行后端测试，确保接口契约、状态机、路径保存和版本保存稳定。
2. 再运行前端 `test:smoke` 和 `typecheck`，确保前端入口、预览沙箱和 API 路径未偏离契约。
3. 配置真实 Tuzi / Claude 环境变量，用“五一劳动节造城者”案例做人工验收。
4. 若人工验收发现视觉质量问题，优先调 PromptAgent 模板，再考虑增强 Review Agent。
